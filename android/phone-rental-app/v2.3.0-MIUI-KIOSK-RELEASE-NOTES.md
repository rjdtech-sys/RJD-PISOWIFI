## What's New

### 🔒 Full-Screen Overlay Kiosk Mode (Works on MIUI/HyperOS!)

This update **solves the Redmi Note 11 Pro 5G problem** where custom launchers cannot be set. Instead of relying on being a launcher, the app now uses **transparent overlays** to block ALL system buttons.

**Key Features:**
- ✅ **Navigation Bar Blocker** - Blocks HOME, Recent Apps, and Back buttons
- ✅ **Status Bar Blocker** - Blocks notification shade pull-down
- ✅ **Dual Overlay System** - Top + Bottom overlays prevent all escape routes
- ✅ **No Launcher Required** - Works without setting as default launcher
- ✅ **MIUI/HyperOS Compatible** - Specifically designed for Xiaomi/Redmi/POCO devices
- ✅ **Universal Solution** - Works on ALL Android devices, not just MIUI

---

## How It Works

### Transparent Overlay System:

The app creates **two invisible overlay windows**:

1. **Top Overlay** (Status Bar)
   - Covers ~80px at the top of screen
   - Blocks notification shade pull-down
   - Prevents access to quick settings

2. **Bottom Overlay** (Navigation Bar)
   - Covers ~120px at the bottom of screen
   - Blocks HOME button
   - Blocks Recent Apps button
   - Blocks Back button
   - Completely transparent - user can't see it

### User Experience:

- ✅ User sees the app normally
- ✅ User can tap buttons and interact with the app
- ✅ User can open whitelisted apps from the grid
- ❌ User **CANNOT** pull down status bar
- ❌ User **CANNOT** tap HOME button (blocked by overlay)
- ❌ User **CANNOT** tap Recent Apps button (blocked by overlay)
- ❌ User **CANNOT** escape the kiosk mode

## Permissions Required

### ⚠️ SYSTEM_ALERT_WINDOW (Overlay Permission)

**This is CRITICAL** for the overlay kiosk mode to work!

**On first launch:**
1. App will request overlay permission
2. Go to **Settings** → **Display over other apps** → Enable for Phone Rental
3. OR: **Settings** → **Apps** → **Phone Rental** → **Advanced** → **Draw over other apps** → Enable

**On MIUI/HyperOS:**
1. Go to **Settings** → **Apps** → **Permissions** → **Other permissions**
2. Find **"Display pop-up windows while running in the background"**
3. Set to **Always allow**

## Troubleshooting

### Problem: Overlays not blocking buttons
**Solution:**
- Check overlay permission is granted
- Go to Settings → Apps → Phone Rental → Permissions → Enable "Display over other apps"
- Restart the app

### Problem: MIUI kills the overlay service
**Solution:**
1. Go to **Settings** → **Battery** → **App battery saver**
2. Find **Phone Rental**
3. Select **"No restrictions"**
4. Enable **Auto-start** for the app

### Problem: User can still access settings via power button
**Solution:**
- This requires Device Owner mode (factory reset needed)
- OR: Use physical button blocker case
- OR: Disable power button in Developer Options (if available)

### Problem: Overlay permission denied
**Solution:**
- On MIUI: Settings → Apps → Permissions → Other permissions → Enable all
- Grant "Display pop-up windows" permission
- May need to grant multiple times on first launch

## Technical Details

### How Overlays Work:

```kotlin
// Top overlay (status bar blocker)
WindowManager.LayoutParams(
    screenWidth,
    statusBarHeight + 8,
    TYPE_APPLICATION_OVERLAY,
    FLAG_NOT_FOCUSABLE or FLAG_NOT_TOUCH_MODAL,
    TRANSLUCENT
).apply {
    gravity = Gravity.TOP
}

// Bottom overlay (navigation bar blocker)
WindowManager.LayoutParams(
    screenWidth,
    navBarHeight + 8,
    TYPE_APPLICATION_OVERLAY,
    FLAG_NOT_FOCUSABLE or FLAG_NOT_TOUCH_MODAL,
    TRANSLUCENT
).apply {
    gravity = Gravity.BOTTOM
}
```

Both overlays:
- Are completely transparent (Color.TRANSPARENT)
- Consume all touch events (return true in OnTouchListener)
- Run as a foreground service (won't be killed by system)
- Restart automatically if killed (START_STICKY)

### What This Update Prevents:
